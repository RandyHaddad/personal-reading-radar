# Digest Quality

A briefing loop should prepare the user's attention, not merely list links.

## Required Per Item

Include:

- title and link;
- source or author;
- publication/upload/issue date, or `date unclear`;
- recency label: `new`, `recent`, `older/canonical`, `catch-up`, or `date unclear`;
- a short gist;
- concrete digestion bullets.

Digestion bullets should name claims, mechanisms, tradeoffs, or questions to watch while reading.

When the user prefers visual or interactive reading, the per-item requirements still apply. Do not let cards, images, HTML, quizzes, or generated visuals replace links, dates, recency, and source clarity.

## Recency

Default to recent material unless the user wants catch-up. Older items need a reason:

- canonical background;
- directly relevant to current work;
- part of a tracked archive;
- newly resurfaced because of current events;
- requested by the user.

Do not bury oldness. Label it.

## Discovery Mix

Do not optimize only for "more of what already worked." A useful loop balances:

- high-confidence items that match known taste;
- adjacent items that stretch the taste model;
- occasional outliers that may reveal new interests;
- catch-up or canonical items when the user has an explicit backlog.

Keep exploration modest and visible. Label exploratory items with the reason they are included. If the user rejects a wildcard, learn from it without overcorrecting the whole profile.

## Verification

Verify links and dates using primary sources when available. For technical, legal, medical, finance, or other high-stakes topics, rely on primary or authoritative sources.

If a date cannot be verified, say `date unclear` and lower confidence unless the item is still obviously useful.

Never invent:

- publication date;
- author;
- source;
- summary of unread content;
- "newness" claims.

## Shape

Avoid rigid newsletter templates unless the user asks for one. Optimize for digestion and taste.

Good style:

```text
The core move is simple: split raw history from curated memory.

- traces are evidence, not memory
- promotion policy is the product
- the memory artifact has to stay inspectable

The thing to watch is where the system decides what deserves to survive into the next run.
```

Bad style:

```text
What to steal:
Why it matters:
Read if:
```

Unless the user likes that structure.

## Cold Start

During calibration:

- keep the digest small;
- avoid overconfident taste claims;
- show uncertainty;
- include varied but coherent candidates;
- invite normal reactions.

## Pre-Delivery Check

Before delivering, verify:

- no casual duplicates;
- every item has a link;
- every item has a date or `date unclear`;
- old items explain why they are included;
- delivery target matches user preference;
- no Kindle or external send happens without the configured approval policy;
- memory updates match file ownership.
